state model

no accounts. no balances. no transaction graph.

an output is a pedersen commitment hiding the amount, a bulletproof showing the math holds, and a sealed note for the owner to read later. a block is valid if inputs minus outputs plus fees equals zero — nothing to update, nothing to route.

spent inputs and outputs cut through each other before they ever land on chain, so history stays small instead of growing forever.

what's actually running

this is what's live on testnet right now, not a roadmap.

01

gossip that hides the source

transactions route through dandelion++ before they're broadcast, so which node first saw a payment isn't easy to tell.

02

block proposals don't stall

if the main proposer misses its slot, any active validator can step in and keep the chain moving.

03

validator set needs no trust

stake registration lives on chain. every node derives the same active validator set independently.

04

rollbacks are clean

a heavier fork shows up, the chain unwinds and replays state without special-casing anything.

05

fees are predictable

you pay for the bytes your transaction takes up. no spikes because the network got busy.

06

talks tcp and websockets

nodes batch-sync aggressively, so bringing a new peer online doesn't take long.

under the hood

one codebase, every target

the core has no native dependencies. the same rust compiles to wasm, android, ios, and desktop.

crypto

ristretto255, bulletproofs, merlin transcripts. no pairing-based crypto anywhere in the stack.

supply

hard capped at 21 billion. most of it enters through block rewards that halve every four years.
