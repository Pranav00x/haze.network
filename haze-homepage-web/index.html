State model

No accounts. No balances. No transaction graph.

An output is the triple (C, π, ξ): a Pedersen commitment C = v·H + r·G on Ristretto255 hiding value v under blinding factor r; a Bulletproofs range proof π certifying v ∈ [0, 2⁶⁴) with no trusted setup; and an optional ChaCha20-Poly1305-sealed note ξ the owner can later decrypt to recover v and r. A kernel is (excess, fee, σ): excess = Σr_in − Σr_out committed to value 0, signed with σ = Schnorr(excess_sk, fee_LE) over a merlin Fiat–Shamir transcript.

Block validity is one identity, checked in core::transaction::validate_with_reward: ΣC_in − ΣC_out − fee·H + reward·H ≟ Σexcess_i, plus every output's range proof and every kernel's signature. There is no account balance to update and no address to route a payment to — just unspent, confidential outputs and the arithmetic that proves a spend is real without revealing how much moved or between which two outputs.

Spent inputs and their matching outputs cut through — cancel out of history entirely, per block and across the whole mempool, before they ever land. A node compacts spent pairs below a rolling horizon (default 1,000 blocks) and still answers any tip-relative UTXO-set query correctly; it just can't replay full history past that point for a peer that asks (PrunedRange, handled via an aggregate-validation fallback instead).

Protocol mechanics

What's actually running on the live testnet right now, not a roadmap.

01

Dandelion++ propagation

20% fluff probability per hop, 15s fallback-fluff timer. The originating node isn't trivially identifiable as the broadcaster — value-privacy extended to the network layer.

02

Weighted proposer selection

proposer(h) = active_validators[H(h ‖ prev_hash) mod |active_validators|]. Any active validator may sign a given height, not just the computed primary — liveness falls through on a timeout instead of stalling.

03

Deterministic validator set

Stake registration is a signed proof-of-ownership op included in blocks (RegisterValidatorOp) — every node derives the same active-validator set by replaying block content, including after a restart.

04

Reorg-safe state

apply_linear_block / rollback_block are exactly symmetric across the UTXO set, kernel-excess set, and every registry — a heavier fork triggers a clean rollback-then-replay.

05

Flat, size-based fees

Mempool floor is FEE_PER_KB · size_bytes / 1000 — no congestion multiplier inflating the default with mempool backlog.

06

Transport-agnostic P2P

TCP and WebSocket peers interoperate transparently. Sync batches 256 blocks at a time, with an aggregate-validation fallback past a peer's compaction horizon.

Under the hood

One codebase, every target

The consensus/crypto core has zero native dependencies and compiles unmodified to WebAssembly and every mobile ABI.

Crypto stack
Ristretto255 (curve25519-dalek), Bulletproofs range proofs, merlin Fiat–Shamir transcripts, ChaCha20-Poly1305 for recoverable output notes, bip39 for seed phrases. Zero pairing-based cryptography anywhere in the tree.
Supply & emission
21,000,000,000 HAZE target: 65% emitted via block rewards, halving every 12,600,000 blocks (~4 years at a 10s block time); 35% minted at genesis (13% team, 13% investors — each vesting over a 6-month cliff plus 7 quarterly tranches — 6% airdrop, 3% treasury).
