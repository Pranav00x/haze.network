Privacy isn’t a feature you bolt on. It has to be the default.

Mimblewimble is a transaction protocol, not a coin — Haze is a from-scratch layer-1 implementation of it. Every value is hidden in a Pedersen commitment, proven non-negative with a Bulletproofs range proof, and every transaction balances via simple additive homomorphism instead of a global ledger of who-owns-what. There's no address book to leak, because there are no addresses — just unspent, confidential outputs.

Spent inputs and their matching outputs can be cut through — cancelled out of history entirely, per block and across the whole mempool — so old, fully-spent transaction chains don't have to be kept around forever just to prove the chain is valid. A node can compact its own history past a rolling horizon and still answer "is this UTXO set correct" for any recent tip.

What runs on it today

06 shipped
01 CORE

Confidential payments, Dandelion-routed

Every payment gossips through a stem phase before it ever fluffs to the wider network, so the node that originates a transaction isn't trivially the first one to broadcast it — the same privacy the base protocol gives values, extended to propagation itself.

02 CORE

Proof-of-stake, deterministic proposer selection

Validators are just currently-unspent UTXOs with a signed proof of ownership — no separate stake-lock token, no slashing complexity. Weighted, deterministic selection per block height means liveness doesn't depend on one designated proposer; an inactive validator's turn falls through to the next automatically.

03 NFT

Native NFTs — no contracts required

Assets are a first-class registry entry, not a token standard implemented in application code. Minting, transferring, and ownership all live in consensus itself.

04 NFT

Scheduled drops with allowlists

Multi-phase collection launches — GTD, FCFS, and public rounds — with Merkle-proven allowlist membership checked directly by consensus, not trusted to an off-chain minting server.

05 MARKET

Trustless secondary sales

Buying and selling settle through an interactive two-party slate: a seller can co-sign a transfer that's cryptographically inert until the buyer's payment kernel actually lands on-chain — no escrow contract, no custodian, no marketplace fee wallet holding funds in between.

06 MARKET

Creator royalties, enforced by consensus

A collection's royalty rate is baked into every resale as a second payment condition alongside the seller's proceeds — the transfer simply can't apply until both land, so there's no royalty-skipping marketplace to route around.

Under the hood

Written to run identically everywhere the same logic is needed — the consensus/crypto core has zero native dependencies and compiles unmodified to WebAssembly and every mobile ABI.

Crypto stack
Ristretto255 (curve25519-dalek), Bulletproofs range proofs, merlin Fiat–Shamir transcripts, ChaCha20-Poly1305 for recoverable output notes. Zero pairing-based cryptography anywhere in the tree.
Network
Transport-agnostic P2P — raw TCP or WebSocket peers interoperate transparently, so a node behind a single-port host can still fully participate. Sync via batched block requests with an aggregate-validation fallback for peers past their compaction horizon.
