A ledger that forgets who paid whom.

Haze is a Mimblewimble L1. No accounts, no address books, no smart contracts. Spent outputs cut through and disappear — so balances and transaction graphs are never public on chain, by design.

state model

No accounts. No balances. No transaction graph.

An output is a Pedersen commitment hiding the amount, a Bulletproof proving the math holds, and a sealed note only the owner can read. There are no addresses to route to and no balances to update — validating a block is one clean equation.

01Amounts are hiddenCommitments and range proofs verify a spend is valid without ever revealing what it's worth.
02The graph is erasedSpent inputs and their outputs cut through — they cancel out before they ever land on chain.
03History stays smallOnly kernels and the current UTXO set are kept; the chain forgets the rest.
block validity, in full
inputs − outputs + fees = 0

That's the whole check. If the commitments balance, the money is conserved — no need to know who, or how much. Everything else the ledger could have remembered simply isn't stored.

shipped, not planned

What's actually running on the network.

Not a roadmap — every mechanic below is live on the public testnet today.

01

Gossip that hides the source

Transactions route through Dandelion++ before broadcast, so which node first saw a payment can't be told apart.

02

Proposals that never stall

If the elected proposer misses its slot, any active validator can step in and keep blocks moving.

03

A trustless validator set

Stake registration lives on chain; every node derives the exact same active set independently.

04

Clean rollbacks

A heavier fork arrives, the chain unwinds and replays state — no special-casing, no corruption.

05

Fees you can predict

You pay for the bytes your transaction takes up. No gas auctions, no spikes when the network is busy.

06

Speaks TCP and WebSocket

Nodes batch-sync aggressively, so bringing a fresh peer online and catching up to the tip is fast.

under the hood

One Rust codebase. Every target.

The core has no native dependencies — the same code compiles to WebAssembly, Android, iOS, and desktop, so every wallet runs the identical consensus logic.

crypto

Ristretto255, Bulletproofs, and Merlin transcripts. Lean by design — no pairing-based crypto anywhere in the stack.

supply

Hard capped at 21 billion HAZE. Most of it enters through block rewards that halve every four years.
