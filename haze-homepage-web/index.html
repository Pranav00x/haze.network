A ledger that doesn’t remember who paid whom.

A from-scratch Mimblewimble implementation: Pedersen-committed, Bulletproofs-range-proved UTXOs with no account model and no address book, settled by a proof-of-stake validator set with deterministic weighted proposer selection. No smart contract layer — consensus rules are compiled into the protocol, not deployed on top of it.

State model

An output is the triple (C, π, ξ): a Pedersen commitment C = v·H + r·G on Ristretto255 hiding value v under blinding factor r; a Bulletproofs range proof π certifying v ∈ [0, 2⁶⁴) with no trusted setup; and an optional ChaCha20-Poly1305-sealed note ξ the owner can later decrypt to recover v and r. A kernel is (excess, fee, σ): excess = Σr_in − Σr_out committed to value 0, signed with σ = Schnorr(excess_sk, fee_LE) over a merlin Fiat–Shamir transcript.

Block validity is one identity, checked in core::transaction::validate_with_reward: ΣC_in − ΣC_out − fee·H + reward·H ≟ Σexcess_i, plus every output's range proof and every kernel's signature. There is no account balance to update and no address to route a payment to — just unspent, confidential outputs and the arithmetic that proves a spend is real without revealing how much moved or between which two outputs.

Spent inputs and their matching outputs cut through — cancel out of history entirely, per block and across the whole mempool, before they ever land. A node compacts spent pairs below a rolling horizon (default 1,000 blocks) and still answers any tip-relative UTXO-set query correctly; it just can't replay full history past that point for a peer that asks (PrunedRange, handled via an aggregate-validation fallback instead).

Protocol mechanics

06 shipped
01 CORE

Dandelion++ payment propagation

Every transaction routes through a stem phase (single random peer-to-peer hop) before fluffing to full broadcast — 20% fluff probability per hop, 15s fallback-fluff timer if the stem stalls. The originating node isn't trivially identifiable as the broadcaster, extending the base protocol's value-privacy to the network layer itself.

02 CORE

Weighted, deterministic proposer selection

proposer(h) = active_validators[H(h ‖ prev_hash) mod |active_validators|]. A validator is any currently-unspent UTXO with a signed proof of ownership — no separate stake-lock token, no slashing. Any active validator (not just the computed primary) may sign a given height, so one offline majority-stake validator can't stall the chain; liveness falls through to the next rank on a timeout instead.

03 CORE

Deterministic validator-set derivation

Stake registration is a signed proof-of-ownership op included in blocks (RegisterValidatorOp), not a live side-channel mutation — every node derives the exact same active-validator set purely by replaying block content, including after a restart, rather than trusting whatever order registrations happened to arrive over the network.

04 CORE

Reorg-safe state transitions

apply_linear_block / rollback_block are exactly symmetric across the UTXO set, the kernel-excess set, and every registry — a fork with more accumulated stake-weight triggers a clean rollback-then-replay to the heavier chain, not a partial or ad-hoc state patch.

05 CORE

Flat, size-based fees

Mempool acceptance floor is FEE_PER_KB · size_bytes / 1000, clamped to a fixed minimum — no congestion multiplier that inflates the default with mempool backlog. A payer can still voluntarily overpay to jump fee-ordered priority queuing; the advisory suggested fee just never auto-escalates on its own.

06 CORE

Transport-agnostic P2P

Raw length-prefixed-bincode TCP and inbound/outbound WebSocket peers interoperate transparently through the same PeerReader/PeerWriter abstraction — a node behind a single exposed HTTP(S) port can still fully participate. Sync batches blocks 256 at a time, falling back to aggregate-kernel validation for any peer serving from past its own compaction horizon.

Under the hood

Written to run identically everywhere the same logic is needed — the consensus/crypto core has zero native dependencies and compiles unmodified to WebAssembly and every mobile ABI.

Crypto stack
Ristretto255 (curve25519-dalek), Bulletproofs range proofs, merlin Fiat–Shamir transcripts, ChaCha20-Poly1305 for recoverable output notes, bip39 for seed phrases. Zero pairing-based cryptography anywhere in the tree.
Supply & emission
21,000,000,000 HAZE target: 65% emitted via block rewards, halving every 12,600,000 blocks (~4 years at a 10s block time); 35% minted at genesis (13% team, 13% investors — each vesting over a 6-month cliff plus 7 quarterly tranches — 6% airdrop, 3% treasury). Every locked-output secret is a real out-of-band scalar, never committed to this repo.
