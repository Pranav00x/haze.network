@haze/agent-sdk — devnet

Payment rails for AI agents, not humans clicking buttons.

An AI agent that pays another agent (or pays for API access, or splits a cost) on a transparent chain exposes its entire counterparty and spending history to everyone it transacts with — every competitor, every customer, every other agent watching the chain. Haze is a Mimblewimble-style chain: balances and transaction graphs are never public on-chain by design. This SDK wraps that in an API shaped for a long-running agent process instead of a wallet UI.

import { HazeAgent } from "@haze/agent-sdk";

const { agent, mnemonic } = await HazeAgent.create({ nodeUrl: "https://your-node" });
// persist agent.export() somewhere durable - the mnemonic is only
// ever available right here, at creation time.

agent.listen(); // starts the background poll loop - required for both
                // sending and receiving to actually complete

agent.on("payment", (p) => {
  console.log(`received ${p.value} from ${p.fromPubkeyHex}`);
});

const txHash = await agent.pay("other-agent.haze", 1_000n);

why a poll loop, not webhooks

Haze has no smart contracts and no push mechanism — agents participate in a two-party handshake over a plain HTTP inbox relay, the same protocol the web and Android wallets already use. listen() runs that handshake for you: it accepts incoming payment requests automatically (no human clicks "approve"), and completes your own outgoing payments once the recipient's agent has responded. Both sides need listen() running for a payment between them to settle.

api

  • HazeAgent.create(opts) — new identity, returns { agent, mnemonic }.
  • HazeAgent.fromMnemonic(mnemonic, opts) — restore from a 12-word phrase.
  • HazeAgent.fromExport(state, opts) — resume a long-running process from agent.export() bytes.
  • agent.pubkeyHex — this agent's public identity.
  • agent.claimName(name) — optional free .haze name, so other agents can pay "you.haze" instead of a raw pubkey.
  • agent.listen() / agent.stopListening().
  • agent.pay(to, amount, { timeoutMs?, fee? }) — resolves with the kernel excess hash once accepted; rejects on timeout.
  • agent.refreshBalance() — { confirmed, pending }.
  • agent.rotateSeed() — generates a fresh identity and sweeps the entire confirmed balance to it in one transaction.
  • agent.on("payment", …) / agent.on("error", …).

install

Not on npm yet — clone the repo and build from source:

git clone https://github.com/Pranav00x/haze
cd haze/agent-sdk
npm install
npm run build

status

This talks to whatever node you point nodeUrl at. Haze is public testnet software — HAZE has no monetary value, and the chain may still be reset. The full protocol (mint, listing, real payment, adversarial rejection paths) is exercised end to end in examples/two-agents.ts against a real devnet node — not a mock.
